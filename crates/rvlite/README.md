# RvLite - Standalone Vector Database

**Status**: Research-tier (v0.3.0)

RvLite is a lightweight, standalone vector database that runs entirely in WebAssembly. It provides SQL, SPARQL, and Cypher query interfaces over a `ruvector-core` vector store with IndexedDB persistence.

> **Note:** graph neural network and self-learning (ReasoningBank) features are **planned, not yet wired** — see [Planned / Not Yet Wired](#-planned--not-yet-wired) below. They are not present in the current API.

## 🎯 Vision

A complete vector database that runs anywhere JavaScript runs:

- ✅ Node.js
- ✅ Deno
- ✅ Bun

- ✅ Vercel Edge Functions

## 🏗️ Architecture

RvLite is a WASM crate that bundles its own SQL, SPARQL, and Cypher
engines (in-crate `sql`, `sparql`, `cypher`, `storage` modules) on top of
the `ruvector-core` vector store:

```
┌─────────────────────────────────────────┐
│  RvLite (src/lib.rs — RvLite, RvLiteConfig)
│  ├─ sql::SqlEngine        (SQL executor) │
│  ├─ sparql::TripleStore   (SPARQL)       │
│  ├─ cypher::CypherEngine  (Cypher)       │
│  ├─ storage::IndexedDBStorage (persist)  │
│  └─ Unified wasm-bindgen API             │
└──────────────┬──────────────────────────┘
               │ depends on
               ▼
┌──────────────────────────────────────────┐
│  ruvector-core (vectors, SIMD, search)   │
│  (default-features = false, memory-only) │
└──────────────────────────────────────────┘
```

The query engines (SQL/SPARQL/Cypher) and IndexedDB storage adapter are
implemented **in this crate**, not pulled from external WASM crates. The
only build dependency on the RuVector side is `ruvector-core`. The
`Cargo.toml` keeps `ruvector-wasm`, `ruvector-graph-wasm`,
`ruvector-gnn-wasm`, `sona`, and `micro-hnsw-wasm` commented out as
future integration points — they are **not** wired in today.

## 🚀 Quick Start

The real WASM API is generated by `wasm-bindgen` from `src/lib.rs`
(`RvLite`, `RvLiteConfig`). Methods are synchronous except the
IndexedDB persistence calls, which return Promises.

```javascript
import init, { RvLite, RvLiteConfig } from './rvlite.js';

await init();

// Create a database (384 dims, cosine by default)
const config = new RvLiteConfig(384);
const db = new RvLite(config);

// Vector insert + search
const id = db.insert([0.1, 0.2, /* ... 384 floats ... */], { label: 'doc-1' });
const results = db.search([0.1, 0.2, /* ... */], 10);

// SQL (pgvector-style syntax)
db.sql(`CREATE TABLE docs (id TEXT PRIMARY KEY, vector VECTOR(384))`);
db.sql(`SELECT id, vector <-> '[...]' AS distance FROM docs ORDER BY distance LIMIT 10`);

// Cypher graph queries
db.cypher(`CREATE (a:Person {name: 'Alice'})-[:KNOWS]->(b:Person {name: 'Bob'})`);

// SPARQL RDF queries
db.add_triple('<http://example.org/a>', '<http://xmlns.com/foaf/0.1/knows>', '<http://example.org/b>');
db.sparql(`SELECT ?s WHERE { ?s <http://xmlns.com/foaf/0.1/knows> ?o }`);

// IndexedDB persistence (Promises)
await db.save();
const restored = await RvLite.load(config);
```

> The `db.gnn.*` (graph neural network) and `db.learning.*`
> (ReasoningBank) examples that previously appeared here are **not
> implemented**. See [Planned / Not Yet Wired](#-planned--not-yet-wired).

## 📦 Current Status (v0.3.0)

Implemented and wired into the public `RvLite` API today:
- ✅ WASM compilation with `ruvector-core` (memory-only)
- ✅ `wasm-bindgen` bindings (`RvLite`, `RvLiteConfig`)
- ✅ Vector insert/search/get/delete (`insert`, `search`, `search_with_filter`, `get`, `delete`)
- ✅ In-crate SQL engine (`sql::SqlEngine`, pgvector-style syntax)
- ✅ In-crate SPARQL triple store + executor (`sparql`)
- ✅ In-crate Cypher engine (`cypher::CypherEngine`)
- ✅ IndexedDB persistence (`save`/`load`/`export_json`/`import_json`)

## 🔭 Planned / Not Yet Wired

These are referenced in the vision/roadmap but are **not present in the
current code** — do not rely on them:

- ⏳ **Graph neural networks** (`db.gnn.*`): no GNN module or
  `ruvector-gnn-wasm` dependency is wired in. `lib.rs` exposes no GNN API.
- ⏳ **Self-learning / ReasoningBank** (`db.learning.*`): no `sona`
  dependency is wired in; there is no learning/training API.
- ⏳ Integration with `ruvector-wasm`, `ruvector-graph-wasm`,
  `micro-hnsw-wasm` (commented out in `Cargo.toml`).
- ⏳ Bundle-size measurement and performance benchmarks.

## 🛠️ Development

### Build

```bash
# Install wasm-pack
curl https://rustwasm.github.io/wasm-pack/installer/init.sh -sSf | sh

# Build for web
cd crates/rvlite
wasm-pack build --target web --release

# Build for Node.js
wasm-pack build --target nodejs --release
```

### Test

```bash
# Run Rust unit tests
cargo test

# Run WASM tests (requires Chrome/Firefox)
wasm-pack test --headless --chrome
wasm-pack test --headless --firefox
```

### Size Analysis

```bash
# Build optimized
wasm-pack build --release

# Check size
ls -lh pkg/*.wasm
du -sh pkg/
```
