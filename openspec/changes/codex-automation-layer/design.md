# Design: Codex automation layer

## Ownership

`.claude` remains source material. `.codex/mirror/.claude` remains byte-for-byte evidence/debug material. The Rust-owned `crates/codex-env` generator compiles the runtime automation view into repo-local `.codex` files and `.agents/skills/codex-*`. Vendor harnesses are not an ownership target.

## Implementation

- Add an automation graph generated by `codex-env` from the same source inventory used for raw mirroring.
- Keep the detailed mirrored Markdown/TOML files available, but make the graph the compact Rust-native capability index.
- Generate agent teams from real configured roles, with deterministic keyword/category selection and compatibility fields (`agents` and `members`).
- Normalize generated `.codex/hooks/*.sh` files for dynamic repository root resolution; do not mutate `.claude/hooks` or the raw mirror.
- Extend doctor/tests so empty/miswired teams, global prompt copying, and stale generated hook paths are caught.

## Non-goals

- No vendor harness migration.
- No destructive edits to `.claude` source material.
- No user-global prompt installation by default.
