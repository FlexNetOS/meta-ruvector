# sevensense-core

[![Crate](https://img.shields.io/badge/crates.io-sevensense--core-orange.svg)](https://crates.io/crates/sevensense-core)
[![Docs](https://img.shields.io/badge/docs-sevensense--core-blue.svg)](https://docs.rs/sevensense-core)
[![License](https://img.shields.io/badge/license-MIT-blue.svg)](../../LICENSE)

> Shared domain primitives for the 7sense bioacoustic intelligence platform.

**sevensense-core** provides the foundational types, traits, and utilities used across all 7sense crates. It defines the core vocabulary of the domain—species identifiers, temporal boundaries, audio metadata, and error types—ensuring consistency throughout the platform.

## Features

- **Strongly-typed identifiers**: UUID newtypes (`SpeciesId`, `RecordingId`,
  `SegmentId`, `EmbeddingId`, `AnalysisId`, `ModelId`) plus the string-based
  `TaxonId` for scientific taxonomy
- **Temporal Primitives**: `TimeRange` (millisecond-based) with overlap and
  containment helpers, `Timestamp`, and `AudioMetadata` duration utilities
- **Value Objects**: `GeoLocation` (with Haversine distance) and validated
  `Confidence` scores
- **Domain Events**: Event types for audit trails and streaming (`domain::events`)
- **Error Handling**: Unified `CoreError` enum with `Io` conversion

## Use Cases

| Use Case | Description | Example Types |
|----------|-------------|---------------|
| Species Management | Track and validate bird species | `SpeciesId`, `TaxonId` |
| Time Handling | Represent audio segments and recordings | `TimeRange`, `AudioMetadata` |
| Audio Metadata | Describe recordings and their properties | `RecordingId`, `AudioMetadata`, `GeoLocation` |
| Error Propagation | Consistent error handling across crates | `CoreError` |

## Installation

Add to your `Cargo.toml`:

```toml
[dependencies]
sevensense-core = "0.1"
```

## Quick Start

```rust
use sevensense_core::{SpeciesId, TimeRange, AudioMetadata};

// Create a strongly-typed (UUID-based) species identifier
let species = SpeciesId::new();
println!("Species: {species}");

// Define a time range (milliseconds from the start of a recording)
let range = TimeRange::new(1_000, 5_000);
println!("Duration: {} ms", range.duration_ms());
```

### UUID-based Identifiers

All `*Id` types in `identifiers` are UUID newtypes generated by the same macro,
so they share the same API:

```rust
use sevensense_core::{SpeciesId, RecordingId};

// New random UUID id
let species = SpeciesId::new();

// From an existing UUID, or parsed from a string
let recording = RecordingId::nil();
let parsed = RecordingId::parse_str("00000000-0000-0000-0000-000000000000").unwrap();

assert_eq!(recording, parsed);
println!("Recording UUID: {}", recording.as_uuid());
```

### Taxon Identifiers

Domain entities use the string-based `TaxonId` for scientific taxonomy:

```rust
use sevensense_core::TaxonId;

// A species-level taxon (genus_species, underscore-separated)
let blackbird = TaxonId::new("Turdus_merula");

assert!(blackbird.is_species());
assert_eq!(blackbird.genus(), Some("Turdus"));
```

### Basic Time Ranges

`TimeRange` is measured in milliseconds from the beginning of a recording.

```rust
use sevensense_core::TimeRange;

// A range from 1s to 5s into the recording
let range = TimeRange::new(1_000, 5_000);

// Duration helpers
println!("Spans {} ms ({} s)", range.duration_ms(), range.duration_secs());

// Check if a point in time (ms) is within range
assert!(range.contains(2_500));
assert!(!range.contains(6_000));
```

### Overlaps and Validation

```rust
use sevensense_core::TimeRange;

let a = TimeRange::new(1_000, 5_000);
let b = TimeRange::new(4_000, 8_000);
assert!(a.overlaps(&b));

// `new` panics on an invalid range; `try_new` returns None instead.
assert!(TimeRange::try_new(5_000, 1_000).is_none());
```

</details>

<details>
<summary><b>Tutorial: Error Handling</b></summary>

### Using Core Errors

`CoreError` is a `thiserror` enum with `InvalidConfig`, `Io`, `NotFound`, and
`Validation` variants. `std::io::Error` converts into it via `From`.

```rust
use sevensense_core::CoreError;

fn process_audio(path: &str) -> Result<(), CoreError> {
    // `?` converts std::io::Error into CoreError::Io automatically
    let _file = std::fs::File::open(path)?;
    Ok(())
}

fn main() {
    if let Err(e) = process_audio("missing.wav") {
        eprintln!("Error: {e}");
    }
}
```

### Constructing Errors

```rust
use sevensense_core::CoreError;

fn validate(input: &str) -> Result<(), CoreError> {
    if input.is_empty() {
        return Err(CoreError::validation("Invalid input"));
    }
    Ok(())
}

let not_found = CoreError::not_found("Recording", "abc-123");
println!("{not_found}");
```

## API Overview

### Core Types

| Type | Description |
|------|-------------|
| `SpeciesId` | UUID-based species identifier (from `identifiers`) |
| `TaxonId` | String-based scientific taxonomy identifier (from `domain`) |
| `RecordingId` | UUID-based recording identifier |
| `TimeRange` | Start/end boundary in milliseconds |
| `GeoLocation` | Geographic coordinates (lat/lon, optional elevation) |
| `AudioMetadata` | Recording metadata (sample rate, channels, format, etc.) |
| `Confidence` | Validated confidence score in `[0.0, 1.0]` |
| `CoreError` | Unified domain error enum |
